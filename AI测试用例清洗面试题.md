# AI 测试用例清洗工程实践题

## 背景

同花顺远航版客户端积累了一批历史测试用例。这些用例来自长期人工维护，质量不完全一致，可能存在操作步骤缺失、预期结果与现状不一致、描述不清、重复、无法执行等问题。

现在希望你使用 AI 辅助完成一个测试用例清洗 demo。

## 材料

面试官会提供以下材料：

- `布局与显示.xls`
- `install.zip`：同花顺远航版客户端安装包

实际业务中，同类用例规模可能达到上万条。样例文件只用于演示和验收 demo 的基本能力。

## 任务

请实现一个可运行的 demo 工程，对输入的测试用例进行清洗，并产出清洗结果。

你可以自行决定：

- 如何理解“清洗”。
- 如何使用 AI。
- 如何组织工程。
- 如何判断清洗结果是否可信。
- 如何让面试官复现你的工作。

## 交付物

请提交一个 git 仓库或压缩包，至少包含：

- 可运行的 demo 工程。
- 项目说明文档。
- 清洗结果文件。
- 结果说明或报告。
- 能体现实现过程的 git 提交记录。

## 要求

- demo 必须能在样例 Excel 上跑通。
- 不接受只有设计文档、没有可运行工程。
- 不接受只把 Excel 内容复制给 AI 后得到一段总结。
- 不接受无法复现的结果。
- 不接受没有 git 历史的最终代码快照。

## 提交说明

请在说明文档中写清楚：

- 如何安装和运行。
- 运行后会生成哪些结果。
- 你在实现中如何使用 AI。
- 当前 demo 的局限性。
